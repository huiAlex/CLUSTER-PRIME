[JBSEAM-780] support message severities in redirect with message <p>It would be nice if redirect directives would allow for message severities to be defined for them.  This happens a lot for scenarios where exceptions cause a redirect.  Instead of the user seeing an INFO message, we want them to see an ERROR or WARN FacesMessage.</p>

<p>For instance, if a severity flag was added to the message, we could do this in pages.xml using the following:</p>

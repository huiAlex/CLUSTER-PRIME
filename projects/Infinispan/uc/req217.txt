[ISPN-370] clear the store before state transfer  <p>Right now, before integrating state (fetch persistent state = true) the cacheStore.clear() is performed within cacheStore. This cacheStore.clear() should be moved outside the store, in StateTransferManager, and state transfer should document that the the store is cleared before integrating remote state. <br/>
This is in relation to: <a href= http://lists.jboss.org/pipermail/infinispan-dev/2010-March/002483.html  class= external-link  rel= nofollow >http://lists.jboss.org/pipermail/infinispan-dev/2010-March/002483.html</a></p>
